## 기획 구상 & 새로운 아키텍처 학습

### 1. 음성 기반 계약 중개 서비스

### 계약서 작성 방법

- 음성을 통한 계약서 작성
  - 실시간 녹음 혹은 녹음 파일을 업로드 할 경우 추출된 정보를 기반으로 표준화된 계약서를 자동으로 작성
- 텍스트 기반 작성 (채팅or챗봇)
- 경우에 따라 템플릿 -> 생성

### 중개 방안

- 사용자 인증 -> 당근 온도 or 계좌 인증 배지
- 계약 성사 확인 및 중개
  - 양측 사용자의 계약 성사 확인 후 최종 승인
  - 중개 서비스를 통해 상호 동의 완료 시 자동으로 입금 처리

### 2. Feature-Sliced Design(FSD) 아키텍처 정리

### 개요

Feature-Sliced Design (FSD)은 대규모 프론트엔드 애플리케이션의 유지보수성과 확장성을 높이기 위한 아키텍처 패턴이다. 계층적 구조를 통해 기능을 모듈화하고, 도메인 중심으로 구성하여 개발 효율성을 극대화한다.

---

### FSD의 주요 계층 구조

```
📦 src/
 ├── 📂 app/          # 애플리케이션 초기화 및 글로벌 설정
 ├── 📂 pages/        # URL과 매칭되는 페이지 컴포넌트
 ├── 📂 widgets/      # 여러 features, entities를 조합한 UI 블록
 ├── 📂 features/     # 독립적인 기능 단위 (로그인, 장바구니 등)
 ├── 📂 entities/     # 주요 도메인 엔티티 (사용자, 상품 등)
 ├── 📂 shared/       # 전역적으로 사용되는 코드 (유틸, API, 공통 UI)
```

---

### 계층별 설명

#### 1️⃣ `app/` - 애플리케이션 초기화 및 글로벌 설정

- 앱의 진입점과 글로벌 설정을 관리한다.
- 주요 구성 요소: `index.tsx`, `routing.tsx`, `providers/`, `config/`, `styles/`

#### 2️⃣ `pages/` - URL 단위 페이지

- 개별 URL과 매칭되는 컴포넌트 집합 (라우트 단위)
- 주요 파일: `index.tsx` (페이지 컴포넌트), `model.ts` (상태 관리), `ui.tsx` (UI 컴포넌트)

#### 3️⃣ `widgets/` - 조합된 UI 블록

- 여러 features 및 entities를 결합하여 만들어지는 UI 단위
- 예: `Header`, `Footer`, `Sidebar`

#### 4️⃣ `features/` - 독립적인 기능 단위

- 개별적인 기능을 구현하는 모듈 (예: 인증, 장바구니, 검색 등)
- 주요 파일: `model.ts` (상태 관리), `ui.tsx` (UI 요소), `api.ts` (서버 요청), `lib.ts` (헬퍼 함수)

#### 5️⃣ `entities/` - 핵심 도메인 엔티티

- 도메인 중심 데이터 모델 관리 (예: 사용자, 상품, 주문 등)
- 주요 파일: `model.ts` (상태 관리), `api.ts` (데이터 요청), `ui.tsx` (UI 요소)

#### 6️⃣ `shared/` - 전역적으로 사용되는 코드

- 여러 곳에서 재사용되는 공통 코드
- 주요 폴더: `api/` (전역 API 클라이언트), `lib/` (유틸 함수), `ui/` (공통 UI 컴포넌트), `config/` (상수 및 환경 변수)

---

### FSD의 주요 개념

#### Layers (레이어 개념)

- FSD는 **기능 중심(FEATURE-FIRST) 아키텍처**이며, 주요 레이어는 다음과 같이 구성된다.
  - **app**: 글로벌 설정 및 앱 초기화
  - **pages**: URL 단위 페이지 관리
  - **widgets**: 조합된 UI 블록 (헤더, 푸터 등)
  - **features**: 독립적인 기능 단위 (인증, 검색, 장바구니 등)
  - **entities**: 주요 도메인 엔티티 (사용자, 상품, 주문 등)
  - **shared**: 공통적으로 사용되는 유틸리티, API, UI 컴포넌트

#### Slices (도메인 단위 그룹)

- 특정 도메인과 관련된 코드들을 여러 레이어에서 관리
- ex. `auth`는 `features/auth/`, `entities/user/`, `shared/api/` 등 여러 곳에 존재할 수 있음

#### Segments (세부 단위 구성)

- 특정 기능을 구성하는 파일 단위
- ex. `features/auth/` 내에 `model.ts`, `ui.tsx`, `api.ts`, `lib.ts` 등이 포함됨

---

#### 정리

- **FSD는 기능 중심(FEATURE-FIRST) 아키텍처**로, 레이어별 역할을 명확히 분리한다.
- **Slices**를 통해 기능을 그룹화하고, **Segments**를 통해 내부를 세분화하여 유지보수성을 높인다.
- 대규모 프론트엔드 애플리케이션에서 **확장성과 코드 조직력**을 강화하는 효과적인 구조이다.
