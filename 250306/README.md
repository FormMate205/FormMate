## 기획 구상 & 기능 체크

### 웹 기반 실시간 딥페이크 탐지

실시간 화상 회의 중 딥페이크 여부를 탐지하는 AI 기반 보안 솔루션

- **목표** : 딥페이크 탐지 AI를 실시간 화상 회의(WebRTC 기반) 시스템에 통합하여 기업 및 금융 사기 방지
- **방법** : 오픈 소스 화상 회의 플랫폼 https://github.com/miroslavpejic85/mirotalk + 딥페이크 탐지 AI 모델 통합
  (실시간 WebRTC 스트림에서 프레임 캡처 후 AI 분석)
- **핵심 기능**
  1. 실시간 딥페이크 탐지: WebRTC 스트림에서 일정 간격마다 프레임을 캡처하여 백그라운드에서 분석 수행
     (Web Worker 또는 서버에서 비동기적으로 분석하여 실시간 영상의 끊김 방지)
  2. 탐지 경고 시스템: 의심스러운 프레임을 감지하면 사용자에게 팝업, 알림 또는 음성 경고 제공
  3. 탐지 기록 저장: 탐지된 딥페이크 프레임을 로그로 저장하여 사후 분석 가능
  4. 다중 탐지 방식: 얼굴 표정, 음성 분석 등 멀티모달 AI 탐지를 적용하여 신뢰성 향상
  5. 웹 기반 통합: 별도의 설치 없이 브라우저에서 바로 실행 가능
     <br>

#### 1. 딥페이크 탐지 AI 모델 (오픈 소스)

오픈소스 모델 활용하여 Kaggle GPU등으로 실행 가능

- https://github.com/iperov/DeepFaceLab → 얼굴 변조 탐지
- https://github.com/ondyari/FaceForensics → 비디오 기반 딥페이크 탐지
- https://github.com/NTech-Lab/deepfake-detection-challenge → Facebook AI 제공

  데이터셋 : https://www.kaggle.com/c/deepfake-detection-challenge

<br>

#### 2. GPU 환경 (무료 GPU : Kaggle GPU)

- TensorFlow Lite, EfficientNet, MobileNet (GPU 최적화)
- 1프레임 분석 속도: **약 0.2~0.5초** (720p 해상도 기준)
- 1분 화상회의 (30FPS → 1800프레임) **분석 시 예상 시간**

- **예상 처리 속도**

  - 모든 프레임 처리: 약 6~15분 (느림)
  - 1초에 1프레임 처리 (~60 프레임): 약 12~30초
  - **5초에 1프레임 처리 (~12 프레임): 약 2~6초** ✅
  - 10초에 1프레임 처리 (~6 프레임): 약 1~3초
    <br>

- **샘플링 간격 별 정확도**
  | 샘플링 간격 | 1분 영상에서 분석할 프레임 개수 | 예상 탐지 정확도 | CPU/GPU 부담 |
  | --- | --- | --- | --- |
  | **모든 프레임 (30FPS)** | 1800프레임 | 🔥 최고 (99% 이상) | 🛑 불가능 (지연 심함) |
  | **1초에 1프레임** | 60프레임 | ✅ 높음 (~95%) | ⚡ GPU 필요 |
  | **5초에 1프레임** | 12프레임 | ⚠ 보통 (~80-90%) | ✅ CPU도 가능 |
  | **10초에 1프레임** | 6프레임 | ❌ 낮음 (~60-70%) | ✅ CPU 가능 |

<br>

- **최적화 전략**
  - 5~10초마다 프레임 샘플링하여 분석 → 실시간 탐지 성능 유지
  - TensorFlow Lite, ONNX Runtime을 활용하여 경량화 및 속도 최적화
