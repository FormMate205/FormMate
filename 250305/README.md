## 기획 구상 & 기능 체크

### 환전 시점 환율 vs. 실시간 환율 비교 시 카드 청구 시점 이슈

- 사용자가 환전한 내역을 가져옴 → `환전 내역 조회 API`
- 현재 환율과 비교 → `환율 조회 API`
- 결제 추천
  - 현재 환율이 더 낮으면 **카드 결제 추천**
  - 환전 시점의 환율이 더 낮으면 **현금 결제 추천**
  - 카드 결제 시, **카드 해외 결제 수수료 포함하여 정확한 금액 비교 →** `카드 결제 API`
  - AR UI에서 "현금이 더 유리합니다" 또는 "카드가 더 유리합니다"를 표시
    **→ 단, 카드 결제는 청구되는 시점의 환율이 적용되므로 고민 필요**
    <br>

#### 해결 방안 구상

<strong>➡ 비교 모델 변경 (환율 변동이 불확실한 카드 결제를 배제)</strong>

- "과거 환전 vs. 현재 환전"
  - 과거에 환전한 현금으로 결제하는 것이 유리한지, 지금 다시 환전하는 것이 유리한지 비교
