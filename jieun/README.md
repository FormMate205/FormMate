## 딥페이크 금융사기 방지 앱(PWA)

### 실시간 딥페이크 탐지 기능

- 사용자 기기로 영상 통화가 걸려올 시 화면 녹화 허용 알림 전송 → 허용 시 화면 녹화 시작 → 통화 종료와 함께 녹화된 영상 분석 (통화 중인 상태에서 분석하면 더 좋음)
- 업로드된 영상 분석도 가능!
- 딥페이크 음성도 감지 → 특정 **사기 패턴(“급히 돈 보내줘” 등)의 키워드 감지** 후 사용자에게 경고 메시지 제공

**실시간 영상 분석 속도 문제**

- 실시간으로 탐지하려면 서버 부하가 커질 수 있음 → **온디바이스 AI(모바일에서 1차 분석 후 서버 요청)** 활용
- Edge AI 기술(예: TensorFlow Lite)로 일부 기능을 기기에서 직접 처리

### 경찰 연락 기능

- 금융사기 감지 시, 경찰에게 연락을 보낼 것인지 알림 제공 → 통화/문자/거부 선택
- 통화: 경찰에게 자동 연락
- 문자: 지정된 내용과 녹화된 영상과 함께 문자 전송 (내용은 고민 좀..)
- 거부: 말그대로 그냥 경찰 연락 거부

### **탐지된 영상/음성 데이터 블록체인 저장 (이력 관리)**

- **아이디어**: 딥페이크 의심 영상이 검출되면 **블록체인에 저장하여 위변조 방지**
- **구현 가능성**:
    - 영상의 해시값을 블록체인에 기록하여 신뢰성 확보
    - 필요 시 금융기관이나 경찰에 증거 자료로 제공 가능
    - **현실적 어려움**: 블록체인 비용(트랜잭션 수수료)과 속도 문제 → 오프체인 저장 + 해시만 기록하는 방식 고려 가능

### **가족/지인 보호 기능 (긴급 알림 전송)**

**📌 아이디어:**

- 딥페이크 의심 탐지 시, **미리 등록한 가족·지인에게 알림 전송**
- 예를 들어 "부모님이 딥페이크 사기에 노출됨"을 가족에게 자동으로 알릴 수 있음

**✅ 구현 가능성:**

- 사용자 연락처 권한이 필요하지만, 기술적으로는 어렵지 않음
- "긴급 모드" 활성화 시 자동 알림 발송 → ~~워치 넣으면 높으신 분들 쓰러짐~~

### **탐지 결과 리포트 제공 (데이터 분석 서비스)**

**📌 아이디어:**

- 사용자가 탐지된 영상을 보고 **"딥페이크 위험 점수"** 확인
- 예: "이 영상은 80% 확률로 조작되었습니다"
- 사용 패턴을 분석해 **개인 맞춤형 금융 보안 리포트 제공**

**✅ 구현 가능성:**

- 딥러닝 기반 분석 기술을 활용하면 가능
- 핀테크 서비스와 연동하면 **사기 위험도 분석 기능**으로 확장 가능
    
## ❌ **현실적인 어려움 & 해결 방법**

### **1️⃣ iOS PWA에서 WebRTC 사용 제한 (가장 큰 문제)**

- **문제:** iOS(PWA 사파리)에서는 WebRTC 기능이 제한적이며, 백그라운드 실행이 불가능
- **해결 방법:**
    - **브라우저에서 실행되도록 유도** → Chrome, Edge 등 WebRTC 지원 브라우저 권장
    - **iOS의 WebRTC 관련 제한을 확인하고 지속적인 업데이트 필요**
    - **최악의 경우**: iOS에서는 영상 분석 기능을 제한하고, **사기 감지 후 경고 메시지 제공 방식으로 차선책 적용**

### **2️⃣ PWA의 백그라운드 실행 & 알림 문제**

- **문제:** iOS에서는 PWA가 백그라운드에서 실행되지 않음 → 실시간 감지가 어려울 수 있음
- **해결 방법:**
    - **Push Notification + 백그라운드 싱크 활용**
    - 영상 통화가 감지되면 **사용자에게 녹화 허용 요청 후 WebRTC 실행**
    - 완전한 실시간 탐지는 어렵지만, **녹화 후 분석 방식으로 보완 가능**

### **3️⃣ 실시간 영상 분석의 성능 문제 (PWA 환경에서 AI 모델 실행 부담 큼)**

- **문제:** PWA에서는 GPU/CPU 성능이 낮아 AI 모델을 실시간으로 돌리기 어렵고, 딥페이크 탐지 속도가 느릴 수 있음
- **해결 방법:**
    - **WebAssembly(WASM) + TensorFlow.js 사용** → 브라우저에서 모델을 실행할 수 있도록 경량화
    - **1차 필터링: Edge AI (클라이언트 측)** → 가벼운 AI 모델로 빠르게 사기 의심 여부 판단
    - **2차 분석: 서버 연산 (딥러닝 모델 적용)** → 고사양 GPU 서버에서 정밀 분석 후 피드백

## 🔥 **결론: PWA로도 충분히 가능! (단, iOS는 제한적)**

- WebRTC를 활용한 **실시간 영상 감지 & 분석 기능은 PWA에서도 구현 가능**
- 다만, iOS에서는 WebRTC/PWA의 제약이 많으므로 **백업 플랜 필요**
- 성능 문제는 **Edge AI + 서버 연산 조합으로 해결 가능**
- 금융사 협업 없이도 **경고 시스템(Push Notification)부터 먼저 구현 가능**

📌 **💡 최적의 전략:**

1️⃣ **MVP는 PWA + WebRTC 기반으로 구현** (WebRTC 스트리밍 + WASM 모델 활용)

2️⃣ **iOS에서는 WebRTC 대신 업로드 후 분석 기능으로 제공**

3️⃣ **성능 최적화를 위해 클라이언트-서버 하이브리드 방식 적용**

**➡ 네이티브 앱이 아닌 PWA로도 충분히 가능하니, 기획 그대로 진행하는 게 좋아 보임! 🚀**
